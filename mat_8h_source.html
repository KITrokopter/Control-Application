00136 /*
00137  * Read the array header for the variable with the specified name from 
00138  * the MAT-file.
00139  * 
00140  * An array header contains all the same information as an
00141  * array, except that the pr, pi, ir, and jc data structures are 
00142  * not allocated for non-recursive data types.  That is,
00143  * Cells, structures, and objects contain pointers to other
00144  * array headers, but numeric, string, and sparse arrays do not 
00145  * contain valid data in their pr, pi, ir, or jc fields.
00146  *
00147  * The purpose of an array header is to gain fast access to 
00148  * information about an array without reading all the array's
00149  * actual data.  Thus, functions such as mxGetM, mxGetN, and mxGetClassID
00150  * can be used with array headers, but mxGetPr, mxGetPi, mxGetIr, mxGetJc,
00151  * mxSetPr, mxSetPi, mxSetIr, and mxSetJc cannot.
00152  *
00153  * An array header should NEVER be returned to MATLAB (for example via the
00154  * MEX API), or any other non-matrix access API function that expects a
00155  * full mxArray (examples include engPutVariable(), matPutVariable(), and 
00156  * mexPutVariable()).
00157  *
00158  * Return NULL if an error occurs.
00159  */
00160 EXTERN_C LIBMAT_API mxArray * matGetVariableInfo(MATFile *pMF, const char * name);
